# motornet.js

> 🤖 **Developed with the help of [Claude](https://claude.com/claude-code).**

A JavaScript port of [**MotorNet**](https://github.com/OlivierCodol/MotorNet) — a toolbox for
simulating biomechanically realistic effectors (point masses and muscled arms) and the neural
networks that control them. This port runs the **forward simulation** and **trained-policy
inference** entirely in the browser, with **no dependencies**.

The original MotorNet is a Python/PyTorch library. This is a faithful re-implementation of its
simulation engine in plain ES modules, validated numerically against the PyTorch reference (see
[Validation](#validation)).

> **Live demo:** <https://adam-matic.github.io/motornetjs/> (or serve the repo locally with
> `npm run serve` and open <http://localhost:8000/demo/>).

## What's included

| Module | Ports | Contents |
| --- | --- | --- |
| `src/skeleton.js` | `motornet.skeleton` | `Skeleton`, `PointMass`, `TwoDofArm` |
| `src/muscle.js` | `motornet.muscle` | `ReluMuscle`, `MujocoHillMuscle`, `RigidTendonHillMuscle`, `RigidTendonHillMuscleThelen`, `CompliantTendonHillMuscle` |
| `src/effector.js` | `motornet.effector` | `Effector`, `ReluPointMass24`, `FreePointMass24`, `Reacher`, `RigidTendonArm26`, `CompliantTendonArm26` |
| `src/environment.js` | `motornet.environment` | `Environment`, `RandomTargetReach` |
| `src/policy.js` | `motornet.policy` | `PolicyGRU` (forward/inference) |
| `src/random.js` | — | Seedable RNG for reproducible noise |

### Scope and differences from the Python library

- **Forward simulation + inference only.** The Python library is differentiable and trains
  networks through the simulation with PyTorch autograd. JavaScript has no autograd engine, so
  this port does **not** train. The intended workflow is: *train in PyTorch → export weights →
  run in the browser* (see [Running a trained policy](#running-a-trained-policy)).
- **Batch size is fixed to 1.** The Python tensors carry a leading batch dimension; this port
  drops it and represents a single simulation. Joint states are flat arrays
  `[pos…, vel…]`; muscle/geometry states are arrays of rows over muscles.
- **float64.** JS uses double precision throughout; the PyTorch reference is float32. The port is
  therefore slightly *more* precise than the reference it is checked against.
- **Noise is not bit-identical to NumPy.** A seedable generator provides reproducible-in-JS noise,
  but a given seed will not reproduce NumPy's exact random stream.

## Quick start

```js
import { RigidTendonArm26, RigidTendonHillMuscle } from 'motornetjs';

const effector = new RigidTendonArm26({ muscle: new RigidTendonHillMuscle() });
effector.reset({ jointState: [0.5, 1.0] });           // shoulder, elbow (radians)

const action = [0.3, 0.0, 0.6, 0.0, 0.2, 0.1];        // 6 muscle excitations in [0, 1]
for (let t = 0; t < 50; t++) effector.step(action);

console.log(effector.states.joint);      // [sho, elb, shoVel, elbVel]
console.log(effector.states.fingertip);  // [x, y] endpoint position
console.log(effector.states.muscle);     // rows: activation, length, velocity, …, force
```

### Using an environment (gym-style)

```js
import { ReluPointMass24, RandomTargetReach } from 'motornetjs';

const env = new RandomTargetReach({ effector: new ReluPointMass24(), maxEpDuration: 1.0 });
let { obs, info } = env.reset();
let res = env.step([0.5, 0.0, 0.0, 0.2]);   // obs, reward, terminated, truncated, info
console.log(res.info.goal, res.info.states.fingertip);
```

### Running a trained policy

Train a `PolicyGRU` in Python with the original MotorNet, then export its weights:

```python
weights = {name: p.detach().numpy().tolist() for name, p in policy.named_parameters()}
json.dump({'input_dim': 12, 'hidden_dim': 32, 'output_dim': 4, 'weights': weights}, open('policy.json', 'w'))
```

Load and run it in the browser:

```js
import { PolicyGRU, ReluPointMass24, RandomTargetReach } from 'motornetjs';

const meta = await fetch('./policy.json').then((r) => r.json());
const policy = new PolicyGRU(meta.input_dim, meta.hidden_dim, meta.output_dim).loadWeights(meta.weights);

const env = new RandomTargetReach({ effector: new ReluPointMass24() });
let { obs } = env.reset();
let h = policy.initHidden();
for (let t = 0; t < 100; t++) {
  const { u, h: nh } = policy.forward(obs, h);
  h = nh;
  ({ obs } = env.step(u, { deterministic: true }));
}
```

The bundled demo (`demo/`) ships with a point-mass reaching policy trained this way
(`demo/policy_pointmass.json`).

## Validation

`test/validate.mjs` replays trajectories generated by the original PyTorch MotorNet
(`test/refs.json`) through the JS port and reports the maximum absolute deviation across joint,
cartesian, geometry, and muscle states, for seven effector/muscle configurations plus the GRU
policy.

```
$ npm test
✓ arm26_rigidhill      first-step=2.49e-4  max=2.49e-4
✓ arm26_thelen         first-step=1.06e-4  max=2.45e-4
✓ arm26_mujoco         first-step=9.76e-5  max=2.99e-4
✓ arm26_compliant      first-step=6.63e-6  max=3.81e-4
✓ relupm24             first-step=1.02e-5  max=3.72e-5
✓ reacher_relu         first-step=2.03e-5  max=7.44e-5
✓ custom_arm_thelen    first-step=6.38e-5  max=1.47e-4
✓ PolicyGRU forward    max=4.85e-8
```

Deviations are at float32 precision after one step; larger end-of-trajectory deviations on the
chaotic arm reflect the float32-vs-float64 difference compounding over the (Euler/RK4) integration,
not a modelling discrepancy.

## Running the demo

```bash
npm run serve     # static server on http://localhost:8000/demo/
```

The demo lets you:
- watch a **trained GRU policy** drive a point mass to reach targets (click to set a target);
- drive an **Arm26** or **point mass** manually with per-muscle activation sliders.
